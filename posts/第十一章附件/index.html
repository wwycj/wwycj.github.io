第十一章附件

基于您上传的课件《第11章 雷达成像.pdf》（特别是关于RDA算法的流程图部分），这是一个非常明确的流程问题。

直接回答您：距离徙动校正 (RCMC) 通常是在 距离向脉冲压缩之后，但在 方位向压缩之前 进行的。

更准确地说，在经典的距离-多普勒算法（RDA）中，它是在 “距离-多普勒域” (Range-Doppler Domain) 进行的。

以下是详细的处理流程和原因分析：

1. 标准处理流程 (基于RDA算法)

请看以下典型的SAR成像步骤：

  1. 原始数据 (Raw Data)：回波信号，未压缩。

  2. 距离向脉冲压缩 (Range Compression)： ← 第一步

    • 操作：对每一个回波脉冲进行匹配滤波。

    • 状态：此时，信号在距离向已经变窄（变成了尖峰），但在方位向还是散开的。此时，距离徙动（弯曲的轨迹）已经显现出来。

  3. 方位向 FFT (Azimuth FFT)：

    • 操作：将数据变换到“距离-多普勒域”。

    • 状态：所有具有相同最近斜距 R0 的目标，其能量分布在同一条弯曲的轨迹上（即 R(fη) 抛物线）。

  4. 距离徙动校正 (RCMC)： ← 这里进行操作

    • 操作：根据多普勒频率 fη 计算偏移量，把弯曲的轨迹“拉直”。

    • 状态：目标的能量被校正到了同一条距离门直线上。

  5. 方位向压缩 (Azimuth Compression)： ← 最后一步

    • 操作：进行方位向的匹配滤波。

    • 状态：将拉直后的能量在方位向聚焦成一个点，得到最终图像。

2. 为什么要在这个顺序进行？

为什么在“距离压缩”之后？

如果在距离压缩之前做 RCMC，数据在距离向上还是一个很长的宽脉冲（比如跨越了几百个采样点）。

  • 此时我们根本分不清哪个信号是属于哪个目标的，能量混叠在一起。

  • 只有先进行距离压缩，把宽脉冲变成窄脉冲（Sinc函数），我们才能清晰地看到一个个“点”在距离门之间移动的轨迹，这时候进行校正（搬移这些点）才是精确和可行的。

为什么在“方位压缩”之前？

方位压缩本质上是把一个目标在不同时间的回波叠加起来（相干累积）。

  • 如果不先做 RCMC，目标的能量分散在不同的距离门里（比如第500行、501行、502行…）。

  • 这时候如果直接做方位压缩（通常是按行进行一维卷积），就会出现“对不准”的情况——你在第500行里找信号，但信号的一半能量跑到了501行里。

  • 这会导致能量无法完全聚焦，成像模糊。

  • 所以必须先拉直（RCMC），确保能量全都在第500行里，然后再压缩。

3. 特殊情况 (CSA算法)

虽然RDA是最直观的例子，但在Chirp Scaling算法 (CSA) 中，顺序略有不同，但逻辑一致：

  • CSA通过相位操作（Chirp Scaling）来“微调”距离徙动。

  • 这个操作发生在距离压缩之前的某个阶段（为了利用未压缩信号的特性），但最终的**“一致RCMC” (Bulk RCMC)** 通常还是结合在距离压缩的相乘步骤中完成的。

总结结论：

在最通用的理解和RDA流程中，先做距离压缩 → 再做RCMC → 最后做方位压缩。
